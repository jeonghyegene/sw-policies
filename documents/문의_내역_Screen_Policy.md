# Screen Policy: 문의 내역

## ■ Screen Meta

* **SCREEN_NAME**: 문의 내역 (마이페이지 > 문의 내역)
* **화면 목적**: 고객이 등록한 문의 내역을 조회하고, 신규 문의를 등록하는 화면
* **화면 형태**: 마이페이지 하위 페이지
* **관련 화면**: 문의 작성 화면, 문의 상세 화면, AS 준수사항 다이얼로그, 자주 묻는 질문 다이얼로그, 접수 목록 다이얼로그

---

## ■ 화면 구조

```
[마이페이지 > 문의 내역]
  ├─ [필터 영역] 기간 / 문의유형 / 상태 / 검색
  ├─ [목록 영역] 문의 목록 테이블
  │     ├─ 문의번호
  │     ├─ 제목 (문의유형 + 문의내용 요약)
  │     ├─ 접수번호
  │     ├─ 작성자
  │     ├─ 작성일
  │     └─ 상태 (답변대기 / 답변완료)
  └─ [문의 등록 버튼]
```

---

## ■ 화면 플로우

```
[문의 내역 목록]
  ├─ [문의 등록] 클릭 → 문의 작성 화면
  └─ [문의 행] 클릭
        ├─ 답변 대기 → 문의 상세 (수정/삭제 가능)
        └─ 답변 완료 → 문의 상세 (수정 불가 / 삭제 가능)
```

---

## ■ Field Rule

---

### [필터 영역]

#### [기간]

* **필수 여부**: 선택
* **선택 항목**: 최근 1개월 / 최근 6개월 / 최근 1년 / 기간 설정
* **기본값**: 최근 1개월
* **UI 정책**: 버튼형 선택, 단일 선택

#### [문의유형]

* **필수 여부**: 선택
* **선택 항목**: 전체 / AS / 고객상담 / 제작 / 출고·배송 / 파일접수 / 회계관련 / 기타 / Q&A
* **기본값**: 전체 선택 (전체 체크)
* **UI 정책**: 다중 선택 (체크박스형)

#### [상태]

* **필수 여부**: 선택
* **선택 항목**: 전체 / 답변 대기 / 답변 완료
* **기본값**: 전체
* **UI 정책**: 버튼형 단일 선택

#### [검색]

* **필수 여부**: 선택
* **입력 규칙**: 자유 텍스트
* **Placeholder**: "검색어를 입력해주세요"
* **UI 정책**: 텍스트 입력 후 검색 실행

---

### [문의 목록 테이블]

#### [표시 항목]

| 컬럼명 | 설명 |
|--------|------|
| 문의번호 | 문의 고유번호 |
| 제목 | [문의유형 카테고리] 문의내용 앞부분 요약 |
| 접수번호 | 연결된 접수번호 (없는 경우 공란) |
| 작성자 | 문의 등록자명 |
| 작성일 | 문의 등록 날짜 (YYYY-MM-DD) |
| 상태 | 답변 대기 / 답변 완료 (뱃지 형태) |

#### [페이지당 표시 건수]

* **기본값**: 20건씩
* **변경 가능**: 10건 / 20건 / 50건 / 100건 선택 가능
* **UI 정책**: 목록 상단 Dropdown으로 선택

#### [페이지네이션]

* **UI 정책**: 하단 페이지 번호 표시

#### [빈 목록 표시]

* **표시 메시지**: "등록한 문의가 없어요"
* **UI 정책**: 목록 영역에 안내 문구만 표시

---

### [문의 등록 버튼]

* **위치**: 필터 영역 우측 상단
* **활성화 조건**: 항상 활성화
* **동작**: 문의 작성 화면으로 이동

---

## ■ 화면 결과

### [목록 행 클릭 시]

* **답변 대기 상태**: 문의 상세 화면으로 이동 (수정/삭제 가능)
* **답변 완료 상태**: 문의 상세 화면으로 이동 (수정 불가 / 삭제 가능)

### [필터 적용 시]

* **처리**: 해당 조건에 맞는 문의 목록만 표시, 총 건수 갱신

---

## ■ 예외 상황

### [검색 결과 없음 시]

* **처리**: "등록한 문의가 없어요" 안내 문구 표시

### [네트워크 오류 시]

* **처리**: 에러 메시지 표시 및 재시도 유도

# Screen Policy: 문의 작성

## ■ Screen Meta

* **SCREEN_NAME**: 문의 작성 (마이페이지 > 문의 내역 > 문의 등록)
* **화면 목적**: 고객이 문의유형을 선택하고 문의 내용을 입력하여 문의를 등록
* **화면 형태**: 마이페이지 하위 페이지
* **관련 화면**: 문의 내역 목록, AS 준수사항 다이얼로그, 자주 묻는 질문 다이얼로그, 접수 목록 다이얼로그

---

## ■ 화면 구조

```
[문의 작성 화면]
  ├─ [자주 묻는 질문] 섹션 (Q&A 유형 제외)
  ├─ [AS 준수사항 안내] (AS 유형 선택 시 필수 확인)
  ├─ [문의유형] 탭 선택
  │     ├─ AS → [세부 카테고리 필수] + [AS 준수사항 확인 체크박스] + [자주 묻는 질문] + [접수번호 필수]
  │     ├─ Q&A → [공개 여부] + [제품 선택]
  │     └─ 그 외 → [세부 카테고리 필수] + [자주 묻는 질문] + [접수번호 선택]
  ├─ [신청자 정보] 이름 (비로그인) / 휴대폰 번호
  ├─ [문의내용] 리치 텍스트 에디터
  ├─ [첨부파일] (Q&A 유형 제외)
  ├─ [개인정보 수집 이용 동의] 체크박스
  └─ [등록하기 버튼]
```

---

## ■ 화면 플로우

```
[문의 등록] 클릭
  ↓
[문의 작성 화면]
  ↓
[등록하기] 클릭
  ↓
AS 유형인 경우: AS 준수사항 확인 여부 체크
  ├─ 미확인 → "AS 준수사항을 체크해주세요" 에러 표시
  └─ 확인 완료 → 필수항목 유효성 체크
        ├─ 실패 → 해당 필드 에러 메시지 표시
        └─ 통과 → 문의 등록 완료 → 문의 내역 화면으로 이동

[뒤로 가기] 클릭 (입력 중)
  ↓
[안내 다이얼로그]
  입력한 내용이 모두 사라져요.
  정말 나가시겠어요?
  [계속 작성하기] [나가기]
  ├─ 계속 작성하기 → 다이얼로그 닫힘, 문의 작성 유지
  └─ 나가기 → 문의 내역 화면으로 이동 (입력 데이터 초기화)
```

---

## ■ Field Rule

---

### [자주 묻는 질문]

* **필수 여부**: 선택 (참고용)
* **노출 조건**: Q&A 유형 제외, 전체 유형에서 노출
* **표시 항목**: Q&A 3건 노출, 클릭 시 답변 펼침/접힘 (Accordion)
* **[전체 보기] 링크**: 클릭 시 FAQ 전체 목록으로 이동

---

### [문의유형]

* **필수 여부**: 필수
* **선택 항목**: AS / 출고·배송 / 제작 / 고객상담 / 파일접수 / 회계관련 / 기타 / Q&A
* **UI 정책**: 탭형 단일 선택, 기본값 AS
* **탭 전환 시**: 세부 카테고리 및 일부 필드 변경 (아래 유형별 상세 참조)

---

### [문의유형별 화면 구성 차이]

| 항목 | AS | Q&A | 그 외 (출고·배송, 제작, 고객상담, 파일접수, 회계관련) | 기타 |
|------|----|-----|------------------------------------------------------|------|
| AS 준수사항 확인 | 필수 (체크박스) | 미노출 | 미노출 | 미노출 |
| 자주 묻는 질문 | 노출 | **미노출** | 노출 | 노출 |
| 공개 여부 | 미노출 | 필수 | 미노출 | 미노출 |
| 접수번호 | 필수 | 미노출 | 선택 | 선택 |
| 제품 선택 | 미노출 | 선택 | 미노출 | 미노출 |
| 세부 카테고리 | 필수 | 미노출 | 필수 | **미노출** |
| 첨부파일 | 노출 | **미노출** | 노출 | 노출 |

---

### [AS 준수사항 확인] (AS 유형 한정)

* **필수 여부**: 필수 (AS 유형 선택 시)
* **UI 정책**: 체크박스 + [AS 준수사항 확인하기] 링크
* **AS 준수사항 확인하기 클릭**: AS 준수사항 다이얼로그 표시
* **미체크 시**: [등록하기] 클릭 시 "AS 준수사항을 체크해주세요" 에러 메시지 표시
* **체크 완료 시**: 체크박스 활성화 상태 유지

#### [AS 준수사항 다이얼로그]

* **내용**: AS 관련 준수사항 전문 (재작업 및 반품 가능기간, 당사 과실, 고객 과실, 환불 및 손해배상, 회수 기준, 회수 방법, 책임 범위 등)
* **[AS 준수사항을 확인했습니다] 버튼**: 클릭 시 다이얼로그 닫힘 + 본 화면 체크박스 자동 체크

---

### [세부 카테고리] (Q&A·기타 유형 제외)

* **필수 여부**: 필수 (Q&A·기타 유형 제외)
* **UI 정책**: Dropdown 선택
* **미선택 시**: "세부 카테고리를 선택해주세요" 에러 메시지 표시

#### [유형별 세부 카테고리 항목]

| 문의유형 | 세부 카테고리 항목 |
|----------|-------------------|
| AS | 미발송 / 상담 / 오배송 / 완제품 이상 / 용지·인쇄·재단·가공·포장 |
| 고객상담 | 고객의 소리 / 자재(입고·성분 등) / 주문자 정보 변경 / 추가주문 / 홈페이지 불편문의 / 회원(기업)가입·정보 수정 |
| 제작 | 담당직원 / 디자인 관련 / 별도견적 / 사이즈 문의 / 색상 문의 / 샘플요청 / 작업 상세 / 작업 수정·취소 / 재접수 / 제작 여부 / 지종 문의 / 포장 |
| 출고·배송 | 배송 메모 / 배송요금 / 배송지 변경 / 선물·후불·방문수령·퀵·월배송 / 제작현황·출고·납기·픽업 문의 |
| 파일접수 | 교정·감리 / 별색 / 성원 웹하드 / 주문 실수·누락·중복 / 파일(작업·수정·취소·폰트·템플릿) |
| 회계관련 | S머니(전환·환불 등) / 가상계좌변경 / 거래명세서·전표 발급 / 서류발급·요청 / 세금계산서 관련 / 입금확인 / 주문(부분)취소 및 환불 / 카드 결제·취소 / 현금영수증 관련 / 환불계좌 등록·변경 |
| 기타 | 없음 |

---

### [공개 여부] (Q&A 유형 한정)

* **필수 여부**: 필수 (Q&A 유형 선택 시)
* **선택 항목**: 공개 / 비공개
* **기본값**: 공개
* **UI 정책**: 버튼형 단일 선택

---

### [제품 선택] (Q&A 유형 한정)

* **필수 여부**: 선택
* **UI 정책**: Dropdown 선택 (제품 카테고리 선택)
* **Placeholder**: "제품을 선택해주세요"

---

### [접수번호]

* **필수 여부**:
  * AS 유형: 필수
  * 그 외 유형 (출고·배송, 제작, 고객상담 등): 선택
  * Q&A 유형: 미노출
* **입력 방식**: 텍스트 직접 입력 또는 드롭다운 선택 (Combo Box)
* **UI 정책**:
  * Placeholder: "접수번호를 입력해주세요"
  * 텍스트 입력 시 하단에 접수 목록 자동 표시 (접수번호 + 제품명 + 접수일)
  * [접수 목록] 버튼 클릭 시: 접수 목록 다이얼로그 표시
* **AS 유형 미입력 시**: "접수번호를 입력해주세요" 에러 메시지 표시

#### [접수 목록 다이얼로그]

* **표시 항목**: 접수번호 / 제품명 (제품유형 - 접수제목) / 접수일
* **검색**: 접수번호 또는 접수제목, 제품명 입력 검색
* **선택 동작**: 항목 클릭 시 다이얼로그 닫힘 + 접수번호 필드 자동 입력

---

### [이름]

* **필수 여부**: 필수 (비로그인 사용자)
* **입력 규칙**: 텍스트 입력
* **Placeholder**: "이름을 입력해주세요"
* **미입력 시**: "이름을 입력해주세요" 에러 메시지 표시
* **로그인 회원**: 미노출 (회원 정보에서 자동 처리)

---

### [휴대폰 번호]

* **필수 여부**: 필수
* **입력 규칙**: 숫자만 입력, "-" 제외
* **길이 제한**: 11자리
* **Placeholder**: "'-'을 제외한 숫자만 입력해주세요"
* **안내 문구**: "답변이 작성되면 해당 번호로 알림을 드려요"
* **로그인 회원**: 가입 시 등록한 번호 자동 입력 (수정 가능)
* **미입력 시**: "휴대폰 번호 11자리를 입력해주세요" 에러 메시지 표시

---

### [문의내용]

* **필수 여부**: 필수
* **입력 규칙**: 자유 텍스트 (리치 텍스트 에디터)
* **길이 제한**: Q&A 유형 최대 500자 / 그 외 유형: 내부 확인 필요사항 참조 (I1)
* **글자 수 표시**: Q&A 유형에서 실시간 글자 수 표시 (0/500)
* **에디터 종류**: 내부 확인 필요사항 참조 (I1)
* **Placeholder**: "문의내용을 입력해주세요"
* **미입력 시**: "문의내용을 입력해주세요" 에러 메시지 표시

---

### [첨부파일] (Q&A 유형 제외)

* **필수 여부**: 선택
* **노출 조건**: Q&A 유형 제외, 전체 유형에서 노출
* **허용 파일 형식**: JPG, PNG, GIF, PDF
* **최대 파일 수**: 3개
* **파일 크기 제한**: 총 20MB 이하
* **UI 정책**:
  * [파일 선택] 버튼 클릭 시 파일 탐색기 열림
  * 첨부 완료 시 파일명 표시 + [X] 삭제 버튼
  * 안내 문구: "파일은 JPG, PNG, GIF, PDF 파일로 최대 3개(20MB 이하)까지 첨부할 수 있어요"
* **허용 형식 외 파일 선택 시**: 에러 메시지 표시 + 파일 자동 제거
* **용량 초과 시**: 에러 메시지 표시 + 파일 자동 제거

---

### [개인정보 수집 이용 동의]

* **필수 여부**: 필수
* **UI 정책**: 체크박스
* **[자세히 보기] 링크**: 클릭 시 개인정보 수집 이용 동의 상세 내용 표시
* **미체크 시**: "개인정보 수집 이용에 동의해주세요" 에러 메시지 표시

### [등록하기 버튼]

* **활성화 조건**: 항상 활성화
* **동작**: 클릭 시 필수 항목 유효성 검증 → 성공 시 문의 등록 완료 → 문의 내역 화면으로 이동
* **유효성 검증 순서**:
  1. AS 유형: AS 준수사항 확인 체크 여부
  2. 세부 카테고리 선택 여부 (Q&A 유형 제외)
  3. 접수번호 입력 여부 (AS 유형)
  4. 이름 입력 여부 (비로그인)
  5. 휴대폰 번호 입력 여부
  6. 문의내용 입력 여부
  7. 개인정보 수집 이용 동의 체크 여부
* **실패 시**: 해당 필드 하단 에러 메시지 표시

# Screen Policy: 문의 상세

## ■ Screen Meta

* **SCREEN_NAME**: 문의 상세 (마이페이지 > 문의 내역 > 문의 상세)
* **화면 목적**: 등록한 문의 내용과 답변을 조회하고, 답변 대기 중인 문의를 수정하거나 문의를 삭제
* **화면 형태**: 마이페이지 하위 페이지
* **관련 화면**: 문의 내역 목록, 문의 수정 화면, 삭제 확인 다이얼로그

---

## ■ 화면 구조

```
[문의 상세]
  ├─ [상단 메타 정보] 문의번호 / 접수번호 / 작성일자 / 작성자 / 휴대폰 번호
  ├─ [문의유형 + 상태 뱃지]
  ├─ [수정 버튼] (답변 대기 상태에서만 노출)
  ├─ [삭제 버튼] (답변 대기 / 답변 완료 모두 노출)
  ├─ [문의내용]
  ├─ [첨부파일] (Q&A 유형 제외)
  ├─ [답변 영역] (답변 완료 시 노출)
  │     ├─ 답변 담당자명
  │     ├─ 답변 작성일
  │     └─ 답변 내용
  └─ [목록 버튼]
```

---

## ■ 상태별 화면 정책

### [답변 대기 상태]

* **수정 버튼**: 노출 → 클릭 시 문의 수정 화면으로 이동
* **삭제 버튼**: 노출 → 클릭 시 삭제 확인 다이얼로그 표시
* **답변 영역**: 미표시

### [답변 완료 상태]

* **수정 버튼**: 미노출 (수정 불가)
* **삭제 버튼**: 노출 → 클릭 시 삭제 확인 다이얼로그 표시
* **답변 영역**: 표시

---

## ■ 표시 항목

### [상단 메타 정보]

| 항목 | 설명 |
|------|------|
| 문의번호 | 문의 고유번호 |
| 접수번호 | 연결된 접수번호 (없는 경우 미노출) |
| 작성일자 | 문의 등록 날짜 (YYYY-MM-DD) |
| 작성자 | 문의 등록자명 |

* **첨부파일**: 파일명 표시, 클릭 시 다운로드 또는 미리보기 (Q&A 유형 제외)

### [답변 영역]

* **표시 조건**: 답변 완료 상태
* **표시 항목**: 답변 담당자명 / 답변 작성일 / 답변 내용

# Screen Policy: 문의 수정

## ■ Screen Meta

* **SCREEN_NAME**: 문의 수정 (마이페이지 > 문의 내역 > 문의 상세 > 수정)
* **화면 목적**: 답변 대기 중인 문의 내용을 수정하여 재등록
* **화면 형태**: 마이페이지 하위 페이지
* **접근 조건**: 답변 대기 상태의 문의만 수정 가능

---

## ■ 화면 구조

```
[문의 수정 화면]
  ├─ 문의 작성 화면과 동일한 구성
  ├─ 기존 입력 내용 자동 로드
  └─ [등록하기 버튼] → 수정 완료 후 문의 내역으로 이동
```

---

## ■ Field Rule

* **문의 작성 화면의 Field Rule과 동일**
* **기존 입력값**: 수정 화면 진입 시 기존 내용 자동 로드
* **수정 가능 항목**: 내부 확인 필요사항 참조 (I2)

---

## ■ 뒤로 가기 / 이탈 정책

* **문의 작성 화면의 이탈 정책과 동일**
* 내용 수정 후 이탈 시 "입력한 내용이 모두 사라져요" 안내 다이얼로그 표시

## ■ 내부 확인 필요사항

### I1. 문의내용 에디터 종류

* **현재 상태**: 내부 확인 필요
* **확인 필요 사항**: 리치 텍스트 에디터 사용 예정이나 구체적인 에디터 라이브러리 미정 (예: Quill, TipTap, Toast UI Editor 등) — 선택된 에디터에 따라 지원 기능(툴바 항목 등) 정의 필요

### I2. 문의 수정 가능 항목 범위

* **현재 상태**: 내부 확인 필요
* **확인 필요 사항**: 수정 시 문의유형 변경 가능 여부, 접수번호 변경 가능 여부 정의 필요
