# Screen Policy: 세금계산서 발급

## ■ Screen Meta

* **SCREEN_NAME**: 세금계산서 발급 (마이페이지)
* **화면 목적**: 출고 완료된 접수건에 대한 세금계산서 신청 및 발급 내역 조회
* **화면 형태**: 단일 페이지 (탭 구조 포함, 다이얼로그 포함)
* **관련 화면**: 마이페이지, 세금계산서 신청서 작성 화면, 세금계산서 신청기업 수정 화면, 나의 고객 목록 다이얼로그, 현금영수증 발급 화면

---

## ■ 화면 구조

```
마이페이지 > 세금계산서 발급
  ├─ [탭1] 신청
  │     ├─ 검색/필터 영역
  │     ├─ 신청 내역 테이블
  │     └─ [버튼] 세금계산서 신청 (다중 선택 시 노출)
  └─ [탭2] 발급 내역
        ├─ 검색/필터 영역
        ├─ 발급 내역 테이블
        └─ [상세 팝업] 발급번호 클릭 시
```

---

## ■ 공통 정책

### [탭 구성]

* **탭1 - 신청**: 세금계산서 신청 가능한 접수건 목록
* **탭2 - 발급 내역**: 발급 완료된 세금계산서 내역 조회

---

### [세금계산서 발행 정책]

#### [자동 발행]

* **시기**: 익월 1일
* **대상**: 수동으로 발행하지 않은 모든 잔여 건 (세금계산서 신청건, 미신청건) 일괄 발행
* **결과**: 회원 정보(기본값)로 강제 발행, 이후 수정 불가

#### [수동 발행]

* **시기**: 출고 완료 후 ~ 익월 1일 자동 발행 전까지 상시 가능
* **방식**: 건별 발행 또는 합산 발행 선택 가능
* **결과**: [신청하기] 버튼 클릭 즉시 발행

#### [합산 발행 기준]

* **기준**: 사업자등록번호
* **작성일**: 선택한 접수건 중 가장 늦은 날짜로 자동 설정

---

### [신청 가능 조건]

| 구분 | 출고 전 | 출고 후 |
|------|---------|---------|
| 미신청건 | 신청 불가 | 익월 1일 전까지 신청 가능 (단, 현금영수증 기신청건은 신청 불가) |
| 세금계산서 신청건 | 신청정보 수정 가능 | 신청정보 수정 가능 |

---

### [증빙 변경 규칙]

| 변경 방향 | 배송 시작 전 | 배송 시작 후 |
|-----------|-------------|-------------|
| 세금계산서 → 현금영수증 | ✅ 변경 가능 | ✅ 익월 1일 전까지 변경 가능 / ❌ 세금계산서 이미 수동 발행된 경우 변경 불가 |
| 현금영수증 → 세금계산서 | ✅ 변경 가능 | ❌ 변경 불가 |
| 미신청 → 세금계산서 | ✅ 신청 가능 | ✅ 신청 가능 (단, 현금영수증 기신청건은 불가) |
| 미신청 → 현금영수증 | ✅ 신청 가능 | ✅ 신청 가능 |

#### [변경 불가 케이스 상세]

* **세금계산서 수동 발행 완료 건**: 현금영수증으로 변경 불가
* **배송 시작 후 현금영수증 신청건**: 세금계산서로 변경 불가
* **배송 시작 후 현금영수증 신청건**: 신청 정보 수정 불가

---

### [대리발급 정책]

* **정의**: 자사 회원 정보가 아닌 타 기업 정보로 세금계산서 발급 (제3자 발행)
* **허용 여부**: 허용
* **방법**: 신청서 작성 시 기업 정보 직접 입력 또는 나의 고객 목록에서 선택

---

### [세금계산서 발급 안내 문구] (탭1 하단 고정)

* 출고 완료 후에 세금계산서 발급을 신청할 수 있어요. 신청하지 않은 건은 익월 1일 이후 자동으로 발급돼요
* 충무로 매장에서 구매한 박스 판매 제품은 충무로 매장 사업자 번호로 세금계산서가 발급돼요
* 폐업 후 배송이 시작된 접수건은 현금영수증 자진발급으로 처리돼요. '마이페이지 > 현금영수증 발급'에서 확인할 수 있어요

### [발급 유형 안내 문구] (탭2 하단 고정)

* 합산: 합산 발행은 여러 접수건을 한 장의 세금계산서로 묶어 처리하는 방식이에요
* 건별: 건별 발행은 한 건의 접수건마다 각각 세금계산서를 발급하는 방식이에요
* 세금계산서를 직접 신청하지 않으면, 다음 달 1일에 자동으로 합산 발급돼요
* 자동으로 묶여 발급된 세금계산서를 건별로 다시 발급받으려면, 국세청 신고 마감일인 다음 달 10일까지만 신청할 수 있어요

## ■ 탭1: 신청

### [검색/필터 영역]

#### [기간 필터]

* **필수 여부**: 선택
* **입력 규칙**: 월별 선택만 가능 (기간 설정 불가)
* **기본값**: 현재 월 (예: 2025년 10월)
* **UI 정책**: 월 단위 Chip 선택

#### [상태 필터]

* **필수 여부**: 선택
* **입력 규칙**: 다중 체크박스 선택
* **선택 항목**: 전체 / 미발급 / 발급대기 / 발급실패
* **기본값**: 전체 선택 (전체 체크)
* **UI 정책**: 개별 체크박스로 복수 상태 동시 필터링 가능

#### [검색어]

* **필수 여부**: 선택
* **입력 규칙**: 자유 텍스트
* **검색 범위**: 전체, 접수번호, 기업명
* **Placeholder**: "접수번호 또는 기업명을 입력해주세요"
* **검색 실행**: 입력 후 Enter 또는 돋보기 아이콘 클릭

---

### [신청 내역 테이블]

#### [컬럼 구성]

| 컬럼명 | 내용 |
|--------|------|
| 체크박스 | 다중 선택용 (신청 가능 건만 체크 가능) |
| 배송시작일 | YYYY-MM-DD 형식 표시 |
| 접수일 | YYYY-MM-DD 형식 표시 |
| 접수번호 | 클릭 시 [접수 상세 모달] 노출 |
| 기업명 | 세금계산서 신청 기업명 |
| 공급가액(₩) | 숫자 표시, 999,999,999원 이상 시 999,999,999+로 표시 |
| 부가세액(₩) | 숫자 표시 |
| 합계금액(₩) | 숫자 표시 |
| 상태 | 상태 뱃지 (미발급/발급대기/발급실패) |
| 비고 | 상태별 보조 설명 (예: 배송 전, 폐기처리) |
| 기능 | [수정] [신청] 버튼 |

#### [상태 정의]

**미발급**

* **미신청건**: 접수 시 증빙 '미신청'한 경우
  * 출고 전: 신청 불가
  * 출고 후: 익월 1일 전까지 신청 가능
  * 단, 현금영수증을 기신청한 미신청건은 세금계산서 신청 불가
* **세금계산서 신청건**: 접수 시 증빙 '세금계산서'를 선택한 경우
  * 출고 전/후 모두 신청정보 수정 가능
* **신청 불가 조건**: 출고 전 미신청건, 현금영수증 기신청 건 (비고에 '배송 전' 표시)
* **폐기처리 표시**: 비고에 '폐기처리' 표시

**발급대기**

* 수동 신청 후 발급 내역 이관 전의 상태
* **수정 완전 불가**: 기업명 포함 모든 항목 수정 불가
* **신청 불가**

**발급실패**

* 정보 오류(폐업 등) 등으로 신청 내역에 남은 실패 상태
* **재신청 가능**: 정보 수정 후 재신청 가능
* **수정 가능**

#### [배송시작일 정책]

* 배송 시작 전부터 표기, 배송 시작 후 'YYYY-MM-DD' 형식 표기

#### [체크박스 선택 정책]

* **신청 불가 건** (출고 전 미신청건, 현금영수증 기신청건): 체크 불가
* **신청 가능 건**: 체크 가능
* **다중 선택 시**: 테이블 상단 우측에 "총 N건 선택됨" 표시 + [세금계산서 신청] 버튼 노출

* **노출 조건**: 모든 행에 노출

  * 출고 전 미신청건
  * 현금영수증 기신청된 미신청건

* **데이터 출처**: 선택한 접수건 합산 자동 계산

#### [발급 유형]

* **필수 여부**: 필수
* **입력 규칙**: 라디오 버튼 선택
* **선택 항목**: 건별 / 합산
* **UI 정책**:
  * 단건 선택 시: 건별 고정 (합산 선택 불가), 안내 문구: "1건의 접수는 '건별' 발급만 가능해요"
  * 다건 선택 시: 건별 / 합산 선택 가능
  * 건별 선택 시 안내: "건별의 경우 선택한 N건의 주문이 각각 1장의 세금계산서로 발급돼요"
  * 합산 선택 시 안내: "'합산'의 경우, 선택한 N건의 주문이 총 1건의 세금계산서로 발급돼요"

#### [작성일]

* **필수 여부**: 자동 설정 (수정 불가)
* **산출 기준**: 선택한 접수건 중 가장 늦은 날짜로 자동 설정
* **폐기처리 건 기준**: 배송시작일이 아닌 폐기처리일 기준 적용
* **안내 문구**: "세금계산서 작성일은 선택한 접수건 중 가장 늦은 날짜로 자동 설정돼요. 폐기된 건은 배송시작일이 아니라 폐기처리일이 기준으로 적용돼요"

## ■ 현업 확인 필요사항

### Q1. 세금계산서 발급 신청 가능 시점 확인

* **현재 상태**: 현업 더블체크 필요
* **확인 필요 사항**: 세금계산서 신청 가능 시점 기준 불일치 — 아래 두 용어는 하루 이상 차이가 날 수 있으므로 어느 기준이 맞는지 확인 필요
  * **배송시작일**: 택배사가 물건을 픽업하여 실제 운송이 시작된 날 (피그마 화면 컬럼명, 증빙정책 원문 기준)
  * **출고완료일**: 성원애드피아 창고/공장에서 물건이 나간 날 (내부 처리 기준)
  * 현재 정책서는 '출고 완료' 기준으로 작성되어 있으나, 증빙정책 원문 및 피그마 화면은 '배송시작일' 기준으로 표기되어 있어 통일 필요
