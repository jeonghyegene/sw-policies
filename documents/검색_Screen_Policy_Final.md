# Screen Policy: 검색

---

## ■ Screen Meta

| 항목 | 내용 |
|------|------|
| SCREEN_NAME | 검색 |
| 화면 목적 | 제품명 또는 키워드를 입력하여 인쇄 제품/상품을 검색하고, 검색 결과를 필터링·정렬하여 조회 |
| 화면 URL | /search |

---

## ■ 화면 구성 (상태별)

| 상태 | 설명 |
|------|------|
| 검색 초기 화면 | 검색 입력창 활성화, 최근 검색어 + 인기 검색어 + 추천 키워드 노출 |
| 검색결과 있을 때 | 검색 필터 + 정렬 + 결과 목록 노출 |
| 검색결과 없을 때 | "검색결과가 없습니다." 메시지 + 관리자 등록 추천 제품 영역 노출 |

---

## ■ Field Rule

### [검색 입력창]
* 필수 여부: 선택
* 입력 규칙: 제품명 또는 키워드 텍스트 자유 입력 (특수문자 입력 차단)
* placeholder: "제품명 또는 키워드를 입력해주세요"
* 길이 제한: 최대 30자
* 실행 조건: 엔터 입력 또는 검색 아이콘 클릭 시 검색 실행
* 미입력 검색 실행 시: 입력창 하단에 "검색어를 입력해 주세요." 안내 메시지 표시
* 특수문자 입력 시: 입력 자체를 차단 (입력 불가 처리)

---

### [최근 검색어]
* 노출 조건: 검색창 포커스 시 (초기 화면)
* 삭제 기능: "모두 삭제" 버튼으로 전체 삭제 가능
* 저장 방식: 로그인/비로그인 모두 브라우저 로컬 저장
* 최대 저장 개수: 5개 (초과 시 가장 오래된 항목부터 자동 삭제)

---

### [인기 검색어]
* 노출 조건: 검색 초기 화면 (검색 전)
* 노출 예시: 최저가, 당일판, 대량인쇄, 디지털, 명함, 소봉투, 카탈로그
* 업데이트 기준: 현업 확인 필요
* 노출 개수: 5개
* 관리 방식: 내부 확인 필요

---

### [추천 키워드]
* 노출 조건: 검색 초기 화면 (검색 전)
* 노출 예시: 명함, 디지털 전단지, 명함 카탈로그, 디지털 명함
* 관리 방식: 내부 확인 필요

---

### [검색 필터 - 카테고리]
* 위치: 검색결과 화면 좌측 필터 영역
* 항목: 검색 결과에 포함된 카테고리만 동적 노출
* "전체" 항목: 검색 결과 카테고리가 2개 이상일 때만 "전체" 항목 함께 노출
* 선택 방식: 단일 선택
* 초기값: "전체" 선택 상태 (카테고리가 1개인 경우 "전체" 미노출, 해당 카테고리만 표시)

---

### [검색 필터 - 용지]
* 위치: 검색결과 화면 좌측 필터 영역
* 항목: 추후 데이터 수급 필요
* 노출 기준: 검색 결과에 포함된 제품에 해당하는 용지 종류만 동적 노출
* 선택 방식: 다중 선택 가능 (체크박스)
* 분류 구분: 스탠다드, 스페셜, 패키지
* 적용 시점: 선택 즉시 자동 반영

---

### [검색 필터 - 오늘판]
* 위치: 검색결과 목록 우측 내용 영역 좌측 상단
* 선택 방식: 체크박스 (토글)
* 적용 시점: 선택 즉시 자동 반영
* 적용 시: "오늘판 신청 가능" 제품만 노출

---

### [필터 초기화 버튼]
* 동작: 현재 선택된 모든 필터 초기화 → "전체" 상태로 복귀
* 노출 조건: 필터 선택 항목이 1개 이상일 때만 노출

---

### [정렬 옵션]
* 위치: 검색결과 목록 우측 내용 영역 좌측 상단
* 항목: 인기순, 낮은가격순, 높은가격순
* 초기값: 인기순
* 인기순 기준: 판매량순
* 선택 방식: 단일 선택, 클릭 즉시 반영

---

## ■ 검색결과 목록 (제품 카드)

| 항목 | 내용 |
|------|------|
| 제품 이미지 | 썸네일 |
| 제품명 | 텍스트 |
| 제품 설명 | 간략 설명 텍스트 |
| 키워드 태그 | 최대 5개, 태그당 최대 6자 (공백 포함) |
| 오늘판 배지 | 썸네일 내 "오늘판" 배지 (해당 시 노출) |
| 간편 접수 배지 | 썸네일 내 "간편 접수" 배지 (해당 시 노출) |
| 최저 가격 | ₩X,XXX 형식 |
| 상세보기 버튼 | 제품 상세 페이지 이동 |

### [키워드 태그 규칙]
* 최대 개수: 5개
* 태그당 최대 글자수: 6자 (공백 포함)
* 초과 시 처리: 현업 확인 필요 (말줄임 처리 / 초과분 미노출)

---

## ■ 스크롤 정책

### [스크롤 구조]
* 좌측 필터 영역과 우측 검색결과 영역은 각각 독립적으로 스크롤

### [검색결과 목록]
* 방식: 일반 스크롤 (무한 스크롤 미적용)
* 사유: 제품 수가 오픈마켓 대비 적으므로 전체 노출 후 스크롤 처리

### [좌측 네비게이션 - 더보기]
* 노출 조건: 용지 또는 카테고리 항목이 10개 이상일 때 "더보기" 버튼 노출
* 동작: "더보기" 버튼 클릭 시 10건씩 추가 노출

---

## ■ 검색결과 없을 때

* 메시지: "검색결과가 없습니다."
* 하단 영역: "지금 가장 인기있는 제품이에요" 문구와 함께 추천 제품 목록 노출
* 추천 제품 관리: 관리자 수동 등록

## ■ 화면 결과

| 시나리오 | 결과 |
|----------|------|
| 검색어 입력 후 결과 있음 | 검색 필터 + 정렬 + 제품 목록 노출 |
| 검색어 입력 후 결과 없음 | "검색결과가 없습니다." + 추천 제품 영역 노출 |
| 검색어 미입력 상태에서 검색 실행 | 입력창 하단에 "검색어를 입력해 주세요." 안내 메시지 표시 |
| 최근 검색어 클릭 | 해당 키워드로 검색 실행 |
| 인기/추천 검색어 클릭 | 해당 키워드로 검색 실행 |
| 용지 필터 선택 | 선택 즉시 검색결과 자동 갱신 |
| 필터 초기화 | 전체 필터 해제 후 검색결과 갱신 |

| 더보기 클릭 | 10건씩 추가 노출 |
| 제품 카드 - 상세보기 | 제품 상세 페이지 이동 |

## ■ 현업 확인 필요사항

* **Q1. 검색 필터 용지 항목 데이터**
  - 필터에 노출할 용지 전체 목록 및 분류(스탠다드/스페셜/패키지) 기준 데이터 수급 필요

## ■ 내부 확인 필요사항

* **Q1. 용지 상세보기 모달 콘텐츠 관리 방식** ← 먼저 확인 필요
  - 모달에 노출되는 썸네일, 설명 등의 콘텐츠가 관리자 직접 등록 형태인지 확인 필요
  - 관리자 등록 형태로 확인 시, 현업 확인 필요사항 Q1(용지 항목 데이터 수급)은 불필요하므로 함께 검토

* **Q2. 키워드 태그 운영 정책**
  - 제품당 등록 가능한 키워드 태그 개수 제한 여부 확인 필요
  - 등록된 태그가 5개 초과 시 노출 방식 확인 필요 (랜덤 5개 노출 / 등록 순서대로 5개 노출 / 전체 노출 등)

* **Q3. 추천 키워드 관리 방식**
  - 운영자 수동 등록/관리 여부 확인 필요

* **Q4. 인기 검색어 관리 방식**
  - 운영자 수동 등록/관리 여부 확인 필요
