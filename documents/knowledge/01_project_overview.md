# 성원애드피아 프로젝트 개요

## 1. 회사 정보
- **회사명**: 성원애드피아
- **업종**: 인쇄 서비스 선도업체
- **연간 매출**: 1,075억 원
- **3년 평균 영업이익**: 109억 원 (약 10% 영업이익률)

## 2. 핵심 비전
- **"궁극의 자동화 구현"**
- 단순 웹사이트 리뉴얼이 아닌 **플랫폼 서비스 공급자(PaaS)** 도약이 목표
- 해외 현지 인쇄업체에 기술 라이선스 및 생산 대행 시스템 구축

## 3. 전략 목표
- 브랜드 가치 강화
- 고객 경험 향상
- 매출 확대 및 업무 효율성 향상
- 데이터 기반 의사결정 체계 구축

## 4. 시스템 구성

| 시스템 | 설명 |
|--------|------|
| 홈페이지 (W2P) | B2B/B2C 통합 인쇄몰 |
| 생산관리시스템 | ERP/MES 통합 |
| 모바일 앱 | 하이브리드 앱 (React Native) |
| 그룹웨어 | 사내 업무 관리 시스템 |

## 5. 기술 환경

| 구분 | 기술 |
|------|------|
| 백엔드 | Java (API 중심) |
| 프론트엔드 | React, Sencha Grid |
| 모바일 | React Native |
| DBMS | MariaDB/MySQL |
| 인프라 | On-Premise, Docker/Kubernetes |
| 폰트 | Pretendard |

## 6. 외부 인터페이스 연동
- PG (토스 PG)
- CTI (고객상담시스템)
- 택배사 API
- 알림톡
- GA4 (분석)
- 국세청 (세금계산서/현금영수증)

## 7. 생산 자동화 연동
- **핫폴더 연동**: 조판 자동화
- **RFID 연계**: 소모품 관리 및 기기 오류 모니터링
- **PITSTOP (아그파)**: 파일 오류 자동 검사/수정

## 8. 용어 표준화 규칙

| 기존 용어 X | 표준 용어 O | 설명 |
|-------------|-------------|------|
| 주문 | **접수** | '주문'은 상품 구매, '접수'는 제품 구매 |
| 주문번호 | **접수번호** | 품목별 부여 (예: 1111-1, 1111-2) |
| 미결제 | **결제 대기** | 결제 미완료 상태 |
| 작업자 | **담당자** | 내부 처리 인력 |
| 최종가 | **현재 단가** | 원자재 단가 등록 시 사용 |
| 직전 | **이전** | 원/부자재 단가 등록 시 사용 |
| 계약서 | **첨부파일** | 단가이력 섹션 컬럼명 |
| 규격 | **단위** | 부자재 등록 화면 |

## 9. 전체 업무 프로세스 개요

### 9.1 End-to-End 흐름
```
[고객 접수] → [파일 검증] → [조판] → [출력/인쇄] → [후가공/재단] → [포장] → [출고/배송] → [정산]
```

### 9.2 접수 프로세스 요약
```
[고객] → 간편/상세주문 → 접수번호 부여(품목별) → 담당자 배정 → 파일 조정 → PITSTOP → 고객 검토 → 생산 진행
```

### 9.3 생산 공정 요약
```
조판 → 출력/인쇄 → 후가공/재단 → 포장 → 출고
```

### 9.4 정산 프로세스 요약
- 세금계산서: **출고 완료 후** 익월 1일 자동 발행 (미신청건 일괄)
- 현금영수증: **배송 시작 후** 발급 (개인 회원 미신청건 → 배송 시작 다음 날 00시 자동 처리)

## 10. 부서별 역할

| 부서 | 역할 |
|------|------|
| **접수팀** | 접수건에 대한 디자인 파일 조정 및 고객 접수 도움 |
| **구매팀** | 자재, 부자재, 소모품에 대한 관리 및 발주 신청 |
| **고객팀** | 고객 CS건 대응 및 상담 |
| **A/S팀** | AS신청건에 대한 분류 및 상담 진행 |
| **생산팀** | 인쇄, 가공(항목별), 재단 등 팀 분리하여 공정별 진행 |
| **재경팀** | 성원 회계 관련 업무 전담 |
| **IT팀** | 성원 내부 개발 진행 |
