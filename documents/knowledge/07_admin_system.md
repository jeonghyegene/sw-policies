# 관리자 시스템

## 1. 대시보드

### 1.1 대표님 통계 우선순위

| 순위 | 항목 | 상세 내용 |
|------|------|----------|
| **1순위** | 매출 및 직원 지출 | 매출액, 직원 카드 지출 금액 (예: 3만 원 이상) |
| **2순위** | 가동률 | 기기별 가동률 현황 |
| **3순위** | 원/부자재 | 원자재/부자재 재고 현황 |

### 1.2 권한 처리
- 권한에 따라 **숫자 마스킹** 처리
- **비활성화/활성화** 처리 필수 적용

### 1.3 생산 현황
- ERP 관련 데이터를 통해 **생산 현황 및 품질 관리** 확인 가능

## 2. 메뉴 IA (대분류)

| 순번 | 1 Depth (대분류) | 주요 기능 |
|------|-----------------|----------|
| 1 | 현황 및 통계 | 대시보드, 접수현황, 회원현황, 생산현황, 회계현황 |
| 2 | 접수 관리 | 접수, AS/수정, 별도견적관리 |
| 3 | 생산 관리 | 조판, 출력/인쇄, 후가공, 자재 |
| 4 | 결제 및 정산 | 결제내역, 세금계산서, 환불, 현금영수증 |
| 5 | 고객 관리 | 회원관리, 1:1 상담 |
| 6 | 배송 및 물류 | 배송 관리 |
| 7 | 시스템 관리 | 시스템 설정 |

## 3. 다국어 지원

### 3.1 지원 언어
- **일본어**
- **영어**

### 3.2 AI 자동 번역 기능
- 관리자 화면에서 **AI 버튼** 제공
- 입력된 **한국어 기반**으로 영어/일본어 **자동 번역**
- 콘텐츠 등록 시 한국어 입력 후 AI 버튼 클릭 → 영어/일본어 자동 생성

## 4. 화면 UX/UI 규칙

### 4.1 기술 스택
- 웹 프론트엔드: **React**
- Grid 컴포넌트: **Sencha Grid**
- 폰트: **Pretendard**

### 4.2 접수 화면
- 접수 상세 화면에서 섹션/정보를 명확하게 구분
- **시각적 구분 강화**

### 4.3 그리드 처리
- 데이터/정보량이 많은 화면에서 **가로 스크롤 옵션 UI** 고려

### 4.4 정보 노출
- '접수현황' → **'업무 이력'**으로 문구 변경

### 4.5 정보 통합
- 기존 탭으로 분리된 주문 상세 정보 (결제, 생산 공정, 배송) **통합**
- 관리자 경험 개선

### 4.6 퀵 메뉴 및 탭 이동
- 우측 상단 퀵 메뉴 **빠른 복원** 기능
- 탭 이동 시 기존 탭 데이터 **저장** 기능 필요

## 5. 권한 체계

### 5.1 권한별 처리

| 권한 수준 | 처리 방식 |
|----------|----------|
| 금액 권한 보유 | 접수 수정 시 승인 없이 직접 수정 가능 |
| 권한 미보유 | 관리자에게 요청 필요 |

### 5.2 대시보드 권한
- 권한에 따라 숫자 마스킹
- 비활성화/활성화 처리

## 6. 주요 관리 기능

### 6.1 접수 관리
- 품목별 접수번호
- 결제 대기 자동 취소 로직
- 수정은 금액 권한자가 승인 없이 직접 수정

### 6.2 생산 관리
- 조판 팀장 승인 (100만 원 이상)
- 용지/품목/수량 기준 합판 선정
- RFID/핫폴더 연동

### 6.3 결제 및 정산
- S머니/S포인트 정책 적용
- '배송완료' 기준 정산 시점 확정
- 폐업 여부 조회

### 6.4 고객 관리
- 등급제 운영
- 휴면 계정 제도 제외
- 탈퇴 간결 처리
- 상담 화면에 주문/결제 정보 노출
