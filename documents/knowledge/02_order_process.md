# 접수 프로세스

## 1. 접수 프로세스 흐름

```
[고객] → 간편주문/상세주문 → 접수번호 부여(품목별) → 담당자 배정 → 파일 조정 → PITSTOP(검증) → 고객 검토(신청건 한정) → 생산 진행(조판)
```

## 2. 접수 경로

### 2.1 간편 주문
- 빠른 접수를 위한 간소화된 주문 방식
- 옵션 변경 필요 시 상세 주문으로 이동 (이정표 방식)
- 간편 주문 내용은 상세 주문으로 자동 이관

### 2.2 상세 주문
- 세부 옵션 설정이 필요한 주문 방식
- 모든 제품 옵션 선택 가능

## 3. 접수번호 체계
- **품목별 부여** 방식 적용
- 예시: 1111-1, 1111-2 (동일 접수 내 품목별 구분)

## 4. 담당자 배정 규칙

| 항목 | 규칙 |
|------|------|
| 자동 배정 | '자동 접수' 버튼 클릭 시 **5건씩** 자동 배정 |
| 최대 배정 | 담당자 1인당 **최대 50건** |
| 동일 업체 | 동일 업체 접수 건은 **동일 담당자에게 자동 배정** |

## 5. 결제 대기 처리
- 결제 대기 건에 고객 알림 발송
- **7일 경과 시 자동 취소** 처리

## 6. 접수 수정/재접수

### 6.1 수정 권한
| 권한 | 처리 방식 |
|------|----------|
| 금액 권한 보유 관리자 | **승인 없이 직접 수정** 가능 |
| 권한 미보유자 | 관리자에게 요청 필요 |

### 6.2 접수 교정 정책
| 금액 기준 | 교정 여부 |
|----------|----------|
| 3만 원 미만 | 불가 |
| 3만 원 이상 | 선택 |
| 20만 원 이상 | 필수 |

### 6.3 파일 재접수
- 1건당 **최대 3회**까지 재접수 가능

## 7. 별도견적 프로세스

### 7.1 정의
- 사이트에 없는 옵션이나, 후불결제용으로 사용하는 접수
- **B2B 고객**이 주로 사용

### 7.2 1.0 → 2.0 주요 변경점

| 항목 | 1.0 (AS-IS) | 2.0 (TO-BE) |
|------|-------------|-------------|
| 신청 주체 | 관리자가 직접 등록 | **고객이 직접 신청** |
| 옵션 처리 | 별도 수동 처리 | 기본 옵션 탑재, 없는 옵션만 별도견적 |
| 결제 방식 | - | 관리자 금액선정 후 **고객 선납금 결제** |

### 7.3 프로세스 흐름
```
[고객] → 접수화면에서 '별도견적' 신청 + 상세내용 입력 → 관리자 검토/금액 선정 → 고객 선납금 결제 (마이페이지) → 생산 진행
```

### 7.4 별도견적 유형별 처리

| 유형 | 처리 방법 |
|------|----------|
| 기존 제품 + 없는 옵션 | 접수화면에서 별도견적 신청 및 상세내용 입력 |
| 카테고리 자체가 없는 제품 | **1대1 게시판**을 통해 견적 안내 → 일반/별도 접수 후 수동 처리 |

### 7.5 화면 구성 요구사항

| 구분 | 페이지 | 필요 기능 |
|------|--------|----------|
| 사용자 | 접수 페이지 | 별도 견적 신청 및 상세 내용 영역, 별도 제품 게시판 문의 안내 영역 |
| 사용자 | 1대1 게시판 | 별도 견적 문의 카테고리 추가 |
| 사용자 | 마이페이지 | 별도 견적건 선납금 결제 창구 및 결제화면 |
| 관리자 | 접수내역 | 별도 견적건 구분, 별도 견적금액 및 선납금 입력 영역 |
| 관리자 | 회원 상세 정보 | 회원 별도 견적 신청 건 대비 주문 횟수 |
| 관리자 | 1대1 문의 게시판 | 별도견적건 필터링 기능 |

### 7.6 확인 필요 사항
- 관리자_생산관리, MES 외 포장 관련 숨겨진 페이지 존재 여부 확인 및 협의 필요

## 8. 파일 검증 (PITSTOP)
- 아그파의 PITSTOP 프로그램 활용
- 파일 오류 자동 검사 및 수정
- 검증 완료 후 생산 단계로 이동

## 9. 고객 검토
- 교정 신청건에 한해 진행
- 고객 확인 후 생산 진행

## 10. 배송지 관리 (마이페이지)

### 10.1 기본 정책
- 로그인 회원 전용 / 마이페이지 내 관리
- 최대 등록 수: **100개** (초과 시 추가 불가)
- 기본 배송지는 1개만 설정 가능 (변경 시 확인 다이얼로그)

### 10.2 검색 기능
- 검색 기준: 전체 / 배송지명 / 주소 / 휴대폰 번호
- 검색 결과 없을 시: "검색 결과가 없어요" 안내

### 10.3 배송지 등록 필드
| 항목 | 필수 여부 | 규칙 |
|------|----------|------|
| 배송지명 | 선택 | 자유 텍스트 |
| 이름(상호명) | **필수** | 한글/영문만 (숫자·특수문자 불가) |
| 휴대폰 번호 | **필수** | 국내 번호 11자리 (010/011 등) |
| 연락처 | 선택 | 숫자 최대 20자리 |
| 우편번호/주소 | **필수** | 우편번호 검색 API 연동 (API 미확정 - 카카오/다음/기타) |
| 상세주소 | 선택 | 우편번호 검색 완료 후 입력 활성화 |
