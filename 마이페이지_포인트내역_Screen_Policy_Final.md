# Screen Policy: 마이페이지 > 포인트 내역

## ■ Screen Meta
* **SCREEN_NAME**: 마이페이지 > 포인트 내역
* **화면 목적**: 회원의 포인트 적립/사용/소멸 이력 조회
* **화면 형태**: 단일 페이지 (필터 + 목록)
* **접근 대상**: 로그인 회원 전용
* **기능 범위**: 조회 전용 (포인트 사용 기능 없음)

---

## ■ 화면 구조

```
[마이페이지 > 포인트 내역]
  ├─ [상단] 보유 포인트 요약
  ├─ [필터] 기간 검색 (Date Picker)
  ├─ [목록] 포인트 내역 리스트
  │     ├─ 날짜
  │     ├─ 구분 (적립 / 사용 / 소멸)
  │     ├─ 내용 (접수번호 포함)
  │     └─ 포인트 (+ / -)
  └─ [페이지네이션]
```

---

## ■ Field Rule

---

### [기간 필터]

#### [기간 입력 방식]
* **UI 유형**: Range Date Picker (시작일 ~ 종료일)
* **기본값**: 최근 1개월 (오늘 기준 1개월 전 ~ 오늘)
* **선택 방식**: 달력 UI에서 시작일과 종료일 범위 선택
* **UI 정책**:
  - 시작일 선택 후 종료일 선택하는 순서로 진행
  - 시작일 > 종료일 설정 불가 (역방향 선택 방지)
  - 선택된 기간 범위는 달력 내 하이라이트 표시
  - 기간 초기화 버튼 제공 (초기화 시 기본값 최근 1개월로 복귀)

#### [빠른 기간 선택 버튼]
* **제공 여부**: 제공
* **선택 항목**: 최근 1개월 / 최근 6개월 / 최근 1년
* **UI 정책**:
  - 버튼 클릭 시 해당 기간으로 Date Picker 자동 설정
  - 기본 선택 상태: 최근 1개월

---

### [포인트 구분 필터]

* **필수 여부**: 선택 (기본값: 전체)
* **선택 항목**: 전체 / 적립 / 사용 / 소멸
* **UI 정책**:
  - 탭 또는 버튼형 단일 선택
  - 선택 시 해당 구분의 내역만 목록에 표시
  - 기간 필터와 구분 필터 동시 적용

---

### [포인트 내역 목록]

#### [목록 표시 항목]
* **날짜**: 포인트 적립/사용/소멸 발생일 (YYYY-MM-DD 형식)
* **구분**: 적립 / 사용 / 소멸
* **내용**: 포인트 발생 사유 (접수번호 포함 시 링크/버튼 형태로 표시)
* **포인트**: 
  - 적립: 양수 표시 (예: +1,000P)
  - 사용/소멸: 음수 표시 (예: -500P)

#### [접수번호 클릭 동작]
* **UI 형태**: 링크 또는 버튼 형태로 표시
* **클릭 동작**: 접수번호 상세 다이얼로그 노출
* **다이얼로그 내용**: 해당 접수 건 상세 정보 표시
* **다이얼로그 닫기**: [닫기] 버튼 또는 외부 영역 클릭 시 닫힘

---

### [페이지네이션]

#### [페이지당 표시 건수]
* **선택 옵션**: 10건 / 20건 / 50건 / 100건
* **기본값**: 20건
* **UI 정책**: Dropdown 방식으로 건수 선택

#### [페이지 이동]
* **UI 유형**: 번호형 페이지네이션
* **이동 방식**: 이전 / 다음 / 페이지 번호 직접 클릭

---

## ■ 보유 포인트 요약 영역

### [표시 항목]
* **보유 포인트**: 현재 사용 가능한 잔여 포인트 (단위: P)
* **소멸 예정 포인트**: 30일 이내 소멸 예정 포인트 및 소멸 예정일 표시
  - 표시 조건: 30일 이내 소멸 예정 포인트가 존재하는 경우에만 노출
  - 표시 예시: "30일 이내 소멸 예정 포인트: 500P (2025-11-30)"

---

## ■ 기능 범위

### [포인트 사용]
* **제공 여부**: 이 화면에서 포인트 사용 기능 없음
* **이유**: 조회 전용 화면
* **포인트 사용 경로**: 결제 화면에서 별도 제공

---

## ■ 세금계산서 발급 정책

### [발급 기준]
* **발급 대상 금액**: 포인트 사용 후 실제 결제 금액에 한하여 발급 가능
* **발급 불가 금액**: 포인트로 차감된 금액은 세금계산서 발급 대상에서 제외
* **예시**:
  - 주문 금액 100,000원 / 포인트 사용 10,000원 / 실 결제금액 90,000원
  - → 세금계산서 발급 가능 금액: 90,000원

---

## ■ 포인트 사용 정책

### [최소 사용 금액]
* **최소 사용 포인트**: 10,000P (임시 확정, 오픈 전 최종 확정 필요)
* **적용 기준**: 1회 결제 시 사용 가능한 최소 포인트 단위

---

## ■ 화면 결과

### [조회 성공 시]
* 설정된 기간에 해당하는 포인트 내역 목록 표시
* 결과 없을 경우: "해당 기간에 포인트 내역이 없어요" 안내 문구 표시

### [접수번호 클릭 시]
* 해당 접수번호 상세 다이얼로그 노출

---

## ■ 예외 상황

### [네트워크 오류 시]
* **처리**: 에러 메시지 표시
* **메시지**: "일시적인 오류가 발생했습니다. 다시 시도해주세요"
* **재시도**: 사용자가 재조회 가능

### [서버 오류 시]
* **처리**: 에러 메시지 표시
* **메시지**: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요"

### [조회 결과 없음 시]
* **처리**: 빈 목록 상태로 표시
* **안내 문구**: "해당 기간에 포인트 내역이 없어요"
