# Screen Policy: 배송지 관리

## ■ Screen Meta
* **SCREEN_NAME**: 배송지 관리
* **화면 목적**: 회원의 배송지 등록/수정/삭제 및 기본 배송지 설정 관리
* **화면 형태**: 마이페이지 내 단일 페이지 + 다이얼로그 (추가/수정/삭제 확인)
* **접근 대상**: 로그인 회원 전용
* **관련 화면**: 배송지 추가 다이얼로그, 배송지 수정 다이얼로그, 삭제 확인 다이얼로그, 기본 배송지 변경 확인 다이얼로그

---

## ■ 화면 구조

```
[배송지 관리] 페이지
  ├─ [검색 영역] 검색 Dropdown + 검색어 입력 필드
  ├─ [목록 상단] 총 N건 / 페이지당 건수 / [+ 배송지 추가] 버튼
  ├─ [배송지 목록] 카드형 리스트
  │     ├─ 체크박스 + 배송지명 + 기본배송지 뱃지
  │     ├─ 이름(상호명) + 우편번호 + 주소 + 상세주소
  │     ├─ 휴대폰 번호 + 연락처(선택)
  │     └─ [수정] [삭제] 버튼
  ├─ [목록 하단] [기본 배송지로 설정] [선택 삭제] 버튼 + 페이지네이션
  └─ [빈 상태] 등록한 배송지가 없을 때 안내 문구
```

---

## ■ 배송지 관리 목록

### [검색 영역]

#### [검색 Dropdown]
* **필수 여부**: 선택 (기본값: 전체)
* **선택 항목**: 전체 / 배송지명 / 주소 / 휴대폰 번호
* **UI 정책**: Dropdown 선택 후 검색어 필드와 연동하여 검색 범위 변경

#### [검색어 입력 필드]
* **필수 여부**: 선택
* **입력 규칙**: 자유 텍스트
* **UI 정책**:
  - Placeholder: "배송지명 또는 주소, 휴대폰 번호 등을 입력해주세요"
  - 검색 아이콘(🔍) 클릭 또는 Enter 키 입력 시 검색 실행
  - 검색 결과 없음 시: "검색 결과가 없어요" 안내 문구 표시

---

### [목록 상단 영역]

#### [총 건수 표시]
* **표시 형식**: "총 N건"
* **갱신 시점**: 배송지 추가/삭제, 검색 실행 시 실시간 갱신

#### [페이지당 건수 Dropdown]
* **기본값**: 20건씩 보기
* **선택 항목**: 20건 / 50건 / 100건 (현업 확인 필요)

#### [+ 배송지 추가 버튼]
* **활성화 조건**: 항상 활성화
* **최대 등록 제한**: 최대 100개 (초과 시 버튼 비활성화 또는 안내 메시지 표시)
* **동작**: 배송지 추가 다이얼로그 노출

---

### [배송지 목록]

#### [배송지 카드 표시 항목]
* **체크박스**: 선택용
* **배송지명**: 등록된 배송지명 (미등록 시 미표시)
* **기본배송지 뱃지**: 기본 배송지로 설정된 경우 표시
* **이름(상호명)**: 수령인 이름
* **주소**: 우편번호 + 도로명 주소 + 상세 주소
* **휴대폰 번호**: 수령인 연락처
* **연락처**: 추가 연락처 (등록된 경우에만 표시)

#### [수정 버튼]
* **활성화 조건**: 항상 활성화
* **동작**: 해당 배송지 정보 자동 입력된 상태로 배송지 수정 다이얼로그 노출

#### [삭제 버튼]
* **활성화 조건**: 항상 활성화
* **동작**: 삭제 확인 다이얼로그 노출

---

### [목록 하단 영역]

#### [기본 배송지로 설정 버튼]
* **활성화 조건**: 체크박스 1개만 선택 시 활성화 (2개 이상 선택 시 비활성화)
* **동작**: 기본 배송지 변경 확인 다이얼로그 노출

#### [선택 삭제 버튼]
* **활성화 조건**: 체크박스 1개 이상 선택 시 활성화
* **동작**: 삭제 확인 다이얼로그 노출

#### [페이지네이션]
* **방식**: 페이지 번호 버튼 클릭 방식 (1, 2, 3...)

---

### [빈 상태]
* **노출 조건**: 등록된 배송지가 없는 경우
* **표시 문구**: "등록한 배송지가 없어요"

---

## ■ Field Rule: 배송지 추가 / 수정 다이얼로그

> 추가 및 수정 다이얼로그는 필드 규칙이 동일하며, 수정 다이얼로그는 기존 데이터가 자동 입력된 상태로 노출됩니다.

---

### [배송지명] (선택)
* **필수 여부**: 선택
* **입력 규칙**: 자유 텍스트
* **길이 제한**: 제한 없음 (현업 확인 필요)
* **UI 정책**:
  - Placeholder: "배송지명을 입력해주세요(선택)"
  - 미입력 시에도 저장 가능

---

### [이름(상호명)]
* **필수 여부**: 필수
* **입력 규칙**: 한글/영문만 허용 (숫자, 특수문자 불가)
* **길이 제한**: 제한 없음 (현업 확인 필요)
* **UI 정책**:
  - Placeholder: "이름(상호명)을 입력해주세요"
  - 미입력 시: "이름을 입력해주세요" 에러 메시지 표시

---

### [휴대폰 번호]
* **필수 여부**: 필수
* **입력 규칙**: 숫자만 입력, "-" 제외, 국내 번호만 허용 (010, 011, 016, 017, 018, 019)
* **길이 제한**: 11자리
* **UI 정책**:
  - Placeholder: "-"을 제외한 숫자만 입력해주세요
  - 미입력 시: "휴대폰 번호를 입력해주세요" 에러 메시지 표시
  - 11자리 미만 시: "휴대폰 번호 11자리를 입력해주세요" 에러 메시지 표시

---

### [연락처] (선택)
* **필수 여부**: 선택
* **입력 규칙**: 숫자만 입력 (하이픈, 특수문자 불가)
* **길이 제한**: 최대 20자리
* **UI 정책**:
  - Placeholder: "연락처를 입력해주세요(선택)"
  - 미입력 시에도 저장 가능

---

### [주소]

#### [우편번호]
* **필수 여부**: 필수
* **입력 규칙**: 자동 입력 (우편번호 검색 API 연동)
* **UI 정책**:
  - 직접 입력 불가 (우편번호 검색 버튼으로만 입력 가능)
  - [우편번호 검색] 버튼 클릭 시 주소 검색 팝업 호출

#### [도로명 주소]
* **필수 여부**: 필수
* **입력 규칙**: 자동 입력 (우편번호 검색 후 자동 반영)
* **UI 정책**: 읽기 전용 (수정 불가)

#### [상세 주소]
* **필수 여부**: 선택 (현업 확인 필요)
* **입력 규칙**: 자유 텍스트 입력
* **길이 제한**: 제한 없음
* **UI 정책**:
  - Placeholder: "상세 주소를 입력해주세요"
  - 우편번호 검색 완료 후 입력 활성화

---

### [기본 배송지로 설정하기]
* **UI 유형**: 체크박스
* **필수 여부**: 선택
* **기본값**: 미체크
* **노출 조건**:
  - **추가 다이얼로그**: 항상 노출
  - **수정 다이얼로그**: 해당 배송지가 이미 기본 배송지인 경우 체크된 상태로 고정 (변경 불가)
* **동작**: 체크 시 저장/수정 완료 시점에 기본 배송지로 설정

---

### [추가하기 / 수정하기 버튼]
* **활성화 조건**: 항상 활성화
* **동작**: 클릭 시 필수 항목 유효성 검증 → 성공 시 저장 처리
* **유효성 검증 순서**:
  1. 이름(상호명) 입력 여부 체크
  2. 휴대폰 번호 형식 및 길이 체크
  3. 우편번호 및 도로명 주소 입력 여부 체크
* **성공 시**: 다이얼로그 닫힘 → 배송지 목록 갱신
* **실패 시**: 해당 필드 하단 에러 메시지 표시

---

### [닫기(X) 버튼]
* **위치**: 다이얼로그 우측 상단
* **동작**: 다이얼로그 즉시 닫힘, 입력 데이터 초기화

---

## ■ Dialog: 삭제 확인

### [화면 구성]
```
삭제 전 안내

배송지를 삭제할까요?
삭제하면 복구할 수 없어요

[아니요]  [삭제하기]
```

### [아니요 버튼]
* **동작**: 다이얼로그 닫힘, 삭제 취소

### [삭제하기 버튼]
* **동작**: 배송지 삭제 완료 → 다이얼로그 닫힘 → 목록에서 해당 배송지 즉시 제거 → 총 건수 갱신

---

## ■ Dialog: 기본 배송지 변경 확인

### [노출 조건]
* 기존 기본 배송지가 있는 상태에서 다른 배송지를 기본 배송지로 설정 시도 시

### [화면 구성]
```
기본 배송지를 변경하시겠어요?

[아니요]  [변경하기]
```

### [아니요 버튼]
* **동작**: 다이얼로그 닫힘, 기존 기본 배송지 유지

### [변경하기 버튼]
* **동작**:
  1. 선택한 배송지 → 기본 배송지로 설정
  2. 기존 기본 배송지 → 자동으로 일반 배송지로 전환
  3. 다이얼로그 닫힘 → 목록 갱신 (기본배송지 뱃지 변경 반영)

---

## ■ 화면 플로우

### [배송지 추가]
```
[+ 배송지 추가] 버튼 클릭
  ↓
[배송지 추가 다이얼로그] 노출 (빈 폼)
  ↓
정보 입력 → [추가하기] 버튼 클릭
  ↓
유효성 검증
  ├─ 성공 → 다이얼로그 닫힘 → 목록 갱신
  └─ 실패 → 에러 메시지 표시, 다이얼로그 유지
```

### [배송지 수정]
```
[수정] 버튼 클릭
  ↓
[배송지 수정 다이얼로그] 노출 (기존 데이터 자동 입력)
  ↓
정보 수정 → [수정하기] 버튼 클릭
  ↓
유효성 검증
  ├─ 성공 → 다이얼로그 닫힘 → 목록 갱신
  └─ 실패 → 에러 메시지 표시, 다이얼로그 유지
```

### [배송지 삭제]
```
[삭제] 버튼 클릭 또는 체크박스 선택 후 [선택 삭제] 클릭
  ↓
[삭제 확인 다이얼로그] 노출
  ├─ [아니요] → 다이얼로그 닫힘, 취소
  └─ [삭제하기] → 삭제 완료 → 목록 갱신
```

### [기본 배송지 설정]
```
체크박스 1개 선택 → [기본 배송지로 설정] 버튼 클릭
  ↓
  ├─ 기존 기본 배송지 없음 → 즉시 기본 배송지로 설정 → 목록 갱신
  └─ 기존 기본 배송지 있음 → [기본 배송지 변경 확인 다이얼로그] 노출
        ├─ [아니요] → 취소
        └─ [변경하기] → 기존 기본 배송지 해제 + 신규 기본 배송지 설정 → 목록 갱신
```

## ■ 예외 상황

### [배송지 최대 등록 초과 시]
* **처리**: [+ 배송지 추가] 버튼 비활성화 또는 클릭 시 안내 메시지 표시
* **메시지**: "배송지는 최대 100개까지 등록할 수 있어요"

### [네트워크 오류 시]
* **처리**: 에러 메시지 표시
* **메시지**: "일시적인 오류가 발생했습니다. 다시 시도해주세요"
* **재시도**: 사용자가 버튼 재클릭으로 재시도 가능

### [서버 오류 시]
* **처리**: 에러 메시지 표시
* **메시지**: "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요"

### [우편번호 검색 팝업 차단 시]
* **처리**: Alert 표시
* **메시지**: "팝업 차단을 해제해주세요"

## ■ 내부 확인 필요사항

### Q1. 우편번호 검색 API 선택
* **현재 상태**: 내부 확인 필요
* **확인 필요 사항**:
  - 카카오 주소 API 사용
  - 다음 주소 API 사용
  - 기타 주소 API 사용

### Q2. 외국인 회원의 배송지 추가 처리
* **현재 상태**: 내부 확인 필요
* **확인 필요 사항**:
  - 외국인 회원도 동일하게 국내 주소만 등록 가능
  - 외국인 회원은 해외 주소 등록 지원 필요 (국가 선택 + 해외 주소 입력 필드 별도 제공)
  - 외국인 회원의 휴대폰 번호 입력 규칙 (국내 번호만 vs 국제 번호 허용)

현재 화면 플로우에서 필요하지만 제공되지 않은 화면:

2. **기본 배송지 변경 확인 다이얼로그** - 기존 기본 배송지 있을 때 변경 시도 시 노출 (디자인 필요)
