## ■ 거래내역

### [필터 및 조회 조건]

| 항목 | 내용 |
|------|------|
| 기간 필터 | 최근 1개월 / 최근 6개월 / 최근 1년 / 기간 설정 (직접 입력) |
| 거래유형 필터 | 전체 (기본값) / 입금 / 사용 / 사용취소 / 출금 / 소멸 (복수 선택 가능) |
| 페이지당 건수 | 20건씩 보기 (Dropdown 변경 가능) |

### [거래내역 테이블 컬럼]

| 컬럼 | 내용 |
|------|------|
| 거래일시 | YYYY-MM-DD HH:MM 형식 |
| 거래유형 | 입금 / 사용 / 사용취소 / 출금 / 소멸 |
| 접수번호 | 사용·사용취소 건에 한해 표시 (예: sw823533) |
| 내용 | 거래 유형별 내용 텍스트 표시 |
| 금액(₩) | +입금 / -차감 형태 |
| 잔액(₩) | 해당 거래 후 잔액 |
| 기능 | 출금 건에 한해 [상세] / [출금 취소] 버튼 표시 |

### [거래유형별 테이블 데이터 표시 정책]

#### 입금 시
| 거래유형 | 접수번호 | 내용 | 금액 | 잔액 | 기능버튼 |
|----------|----------|------|------|------|----------|
| 입금 | - (미표시) | 입금 완료 | +금액 | 잔액 | 없음 |

#### 출금 시
출금 신청 후 상태 변화에 따라 내용 컬럼이 업데이트됨

| 상태 | 거래유형 | 내용 | 금액 | 잔액 | 기능버튼 |
|------|----------|------|------|------|----------|
| 출금 신청 직후 | 출금 | 출금 대기 | -금액 | - (미표시) | [상세] [출금 취소] |
| 출금 성공 시 | 출금 | 출금 완료 | -금액 | - (미표시) | [상세] |
| 출금 실패 시 | 출금 | 출금 실패<br>실패 사유: 정보가 올바르지 않습니다. | -금액 | - (미표시) | [상세] [출금] |

> **UI 정책**: 출금 건의 잔액 컬럼은 미표시 (출금 처리는 실시간이 아닌 배치 처리이므로 잔액 확정 전 표시 불필요)
> **출금 실패 시**: 실패 사유 텍스트를 내용 컬럼 하단에 표시 + [출금] 버튼으로 재출금 신청 가능

#### 사용 시
| 거래유형 | 접수번호 | 내용 | 금액 | 잔액 | 기능버튼 |
|----------|----------|------|------|------|----------|
| 사용 | sw823533 | 제품 접수 | -금액 | 잔액 | 없음 |

#### 사용취소 시
| 거래유형 | 접수번호 | 내용 | 금액 | 잔액 | 기능버튼 |
|----------|----------|------|------|------|----------|
| 사용취소 | sw823533 | 접수 취소 | +금액 | 잔액 | 없음 |

> **사용취소 내용 유형**: 전체취소 환불 / 부분취소 환불

#### 소멸 시
| 거래유형 | 접수번호 | 내용 | 금액 | 잔액 | 기능버튼 |
|----------|----------|------|------|------|----------|
| 소멸 | - (미표시) | 소멸기한 만료 | -금액 | 잔액 | 없음 |

### [거래유형별 발생 케이스 정의]

| 유형 | 발생 케이스 | 내용 텍스트 |
|------|------------|------------|
| **입금** | 제품 접수 전 가상계좌 선입금 | 입금 완료 |
| **입금** | 제품 접수 후 고객이 가상계좌로 입금 | 입금 완료 |
| **사용** | 제품 접수 시 결제금액 차감 | 제품 접수 |
| **사용** | 기접수건 자동 차감 | 제품 접수 |
| **사용취소** | 접수 전체 취소/환불 | 전체취소 환불 |
| **사용취소** | 부분취소 환불 | 부분취소 환불 |
| **출금** | 고객 요청에 따른 출금 | 출금 대기 → 출금 완료 / 출금 실패 |
| **소멸** | 소멸 시효 만료에 따른 예치금 소멸 | 소멸기한 만료 |

### [[상세] 버튼 정책]

* **노출 조건**: 출금 유형 거래 건에만 [상세] 버튼 노출
* **동작**: 마우스 호버 시 툴팁 형태로 출금 상세 정보 표시
* **표시 항목**:

| 항목 | 내용 |
|------|------|
| 출금자 | 예금주명 (예: 정길동) |
| 출금 은행 | 은행명 + 계좌번호 (예: 국민 23444477999998) |
| 출금 사유 | 사용자가 입력한 출금 사유 텍스트 |

### [[출금 취소] 버튼 정책]

* **노출 조건**: 출금 상태가 "출금 대기"인 건에만 노출
* **노출 제거 조건**: 출금 완료 / 출금 실패로 상태 변경 시 버튼 미노출
* **취소 가능 기간**: 출금 신청 후 익일 오후 2시 30분까지
* **주말/공휴일 특례**: 금·토·일 신청 건은 월요일 오후 2시 30분에 처리
* **취소 완료 시**: 출금가능 잔액으로 즉시 복원

## ■ Field Rule

### [가상계좌 변경]

| 항목 | 내용 |
|------|------|
| 진입 경로 | 성원머니 메인 > [가상계좌 변경] 버튼 클릭 |
| 표시 정보 | 현재 예금주명 (읽기 전용, 수정 불가) |
| 입력 항목 | 은행 선택 |
| 은행 선택 UI | 버튼형 그리드 (최대 6개 기본 노출) |
| 이미 등록된 은행 | 비활성화 처리 (선택 불가) |
| 버튼 활성화 조건 | 현재 등록된 은행과 다른 은행 선택 시 [변경하기] 버튼 활성화 |
| 변경 제한 안내 | "가상계좌 입금 은행을 변경하신 경우, 변경일로부터 한 달간은 재변경이 어려워요" |
| 변경 성공 처리 | 다이얼로그 닫힘 + 성원머니 페이지 새로고침 |
| 변경 실패 처리 | 실패 안내 메시지 표시 |

#### 가상계좌 변경 시 은행 목록 (기본 노출)
* 국민은행, 우리은행, 신한은행, IBK기업은행, NH농협은행, 수협은행, 하나은행
* **비활성화**: 현재 등록된 은행은 선택 불가 상태로 표시

### [출금 신청]

| 항목 | 내용 |
|------|------|
| 진입 경로 | 성원머니 메인 > [출금] 버튼 클릭 |
| 버튼 활성화 조건 | 성원머니 출금가능 잔액 > 0원일 경우 [출금] 버튼 활성화 |
| 환불계좌 미등록 시 | "등록된 환불계좌가 없어요" 안내 다이얼로그 표시 → [환불계좌 등록하기] 버튼 제공 |
| 환불계좌 등록 시 | 출금 신청 화면으로 진행 |
| 출금 한도 | 1회 한도 없음 / 1일 한도 없음 (상한선 무제한) |

#### 출금 가능 잔액 계산

| 항목 | 내용 |
|------|------|
| 계산 방식 | 보유 잔액 - 출금대기 금액 = 출금 가능 잔액 |
| 예시 | 보유 150,000원 / 출금대기 20,000원 → 출금가능 130,000원 |
| 초과 입력 시 | 입력 금액이 출금가능 금액 초과 시 → 출금가능 금액으로 자동 입력 처리 |

#### 출금 금액 빠른 선택

| 버튼 | 기능 |
|------|------|
| 전액 | 출금가능 잔액 전액 자동 입력 |
| +100만 | 현재 입력 금액에 1,000,000원 추가 |
| +10만 | 현재 입력 금액에 100,000원 추가 |
| +5만 | 현재 입력 금액에 50,000원 추가 |
| +1만 | 현재 입력 금액에 10,000원 추가 |

#### 출금 사유

| 항목 | 내용 |
|------|------|
| 선택지 | 회계/정산 / 잔액 정리 / 직접 입력 |
| 직접 입력 선택 시 | 사유 입력 Textarea 표시 (최대 50자) |
| 사유 용도 | 내부 확인 및 정산 참고용으로만 사용 (외부 공개 없음), [상세] 버튼 호버 시 출금 사유 표시 |

### Q1. 해외 금융기관 추가 여부
* **현재 상태**: 현업 확인 필요
* **확인 필요 사항**:
  - 현재 환불계좌 변경 은행 목록은 국내 금융기관만 지원
  - 해외 금융기관(외국계 은행, 해외 계좌 등) 추가 지원 여부 결정 필요
  - 추가 시: 지원 국가 범위, 은행 목록, UI 노출 방식 (별도 탭 또는 기존 목록 통합) 정의 필요

* **현재 상태**: 현업 확인 필요
