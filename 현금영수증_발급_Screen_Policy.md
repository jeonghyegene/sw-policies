# Screen Policy: 현금영수증 발급

## ■ Screen Meta
* **SCREEN_NAME**: 현금영수증 발급
* **화면 목적**: 성원페이 결제 주문에 대한 현금영수증 신청 및 발급 내역 조회
* **화면 형태**: 단일 페이지 (탭 구조: 신청 / 발급 내역)
* **진입 경로**: 마이페이지 > 현금영수증 발급
* **접근 대상**: 로그인 회원 전용
* **회원 유형별 구분**: 개인 회원 / 기업 회원 (용도구분 선택지 상이)

## ■ 공통 이용 안내 문구

화면 상단 고정 표시 (수정 불가):

* "현금영수증은 국세청 회원가입 여부와 관계없이 발급돼요. 다만 소득공제 혜택을 받으려면 국세청 홈택스에 회원가입을 해야 해요"
* "현금영수증 서비스 등록이나 자세한 내용은 국세청 홈택스(www.hometax.go.kr)에서 확인할 수 있어요"
* "자진발급한 현금영수증은 발급 내역에서 바로 확인할 수 있어요"

## ■ 상태 설계 배경

현금영수증 상태를 **발급 상태**와 **신청 상태** 두 축으로 분리한 이유는 접수 시점과 배송 시점이 다르기 때문이다.

고객은 접수 시 증빙 수단을 미리 선택(신청완료)하거나, 선택하지 않은 채(미신청) 접수할 수 있다. 발급은 배송 시작 이후에만 이루어지므로, 같은 **미발급** 상태라도 아래 두 경우가 혼재한다.

* 접수 시 현금영수증을 선택했지만 아직 배송이 시작되지 않은 건 → 신청완료 / 미발급
* 접수 시 현금영수증을 선택하지 않아 배송 후 별도 신청이 필요한 건 → 미신청 / 미발급

발급 상태 단일 컬럼으로는 이 두 경우를 구분할 수 없어, 목록에서 **신청 버튼**과 **수정 버튼** 노출 여부를 제어하기 위해 신청 상태를 별도 축으로 분리하였다.

| 발급 상태 | 신청 상태 | 의미 | 액션 |
|----------|----------|------|------|
| 미발급 | 미신청 | 접수 시 미선택, 아직 신청 전 | **신청** 버튼 노출 |
| 미발급 | 신청완료 | 접수 시 선택, 배송 전 | **수정** 버튼 노출 |
| 발급대기 | 신청완료 | 배송 시작, 발급 처리 중 | **수정** 버튼 노출, 클릭 시 정보 수정 불가 안내 (읽기 전용) |
| 발급실패 | 신청완료 | 정보 오류로 발급 실패 | **신청** 버튼 노출 (재신청) |

## ■ 화면 플로우

### [신청 탭 - 신청 플로우]
```
[마이페이지 > 현금영수증 발급 > 신청 탭]
  ↓
[기간/발급상태/신청상태 필터 설정]
  ↓
[목록 조회]
  ├─ 신청 가능 건 → [신청] 버튼 클릭
  │     ↓
  │  [Dialog] 현금영수증 신청
  │     ↓
  │  용도구분 선택 + 발급번호 입력
  │     ↓
  │  [신청하기] 클릭 → 신청 완료 → 목록 갱신
  │
  ├─ 수정 가능 건 → [수정] 버튼 클릭
  │     ├─ 배송 전 → [Dialog] 수정 가능 화면 → 정보 수정 → [수정하기]
  │     └─ 배송 후 → [Dialog] 수정 불가 안내 화면 (읽기 전용)
  │
  └─ 접수번호 클릭 → [Dialog] 접수 상세(전체)
```

### [발급 내역 탭 플로우]
```
[마이페이지 > 현금영수증 발급 > 발급 내역 탭]
  ↓
[기간 필터 선택]
  ↓
[발급 목록 조회]
  ├─ [영수증 인쇄] 클릭 → 브라우저 인쇄 기능 실행
  └─ [자진발급 정보] 클릭 → [Dialog] 자진발급 정보 팝업
```

## ■ 예외 상황

### [신청할 내역 없음]
* **처리**: "신청할 현금영수증 내역이 없어요" 안내 문구 표시

### [발급 내역 없음]
* **처리**: "발급 내역이 없습니다" 안내 문구 표시

### [신청 마감일(다음 달 1일) 초과 시]
* **처리**: 신청 불가 처리
* **안내**: CS 문의로 안내

### [폐업자 발급 시도 시]
* **수동 발행 시**: 발급실패 처리
* **자동 발행 시**: 발급 실패 처리 후 자진발급(현금영수증)으로 자동 전환

### [폐기처리/AS건 (부분취소 후 잔여금액 존재 시)]
* **처리**: 관리자 부분취소 처리 후 차감하고 남은 금액이 0원 이상인 경우 정상 발급 가능

### [네트워크 오류 시]
* **처리**: 에러 메시지 표시
* **재시도**: 사용자가 다시 버튼 클릭하여 재시도 가능

### [서버 오류 시]
* **처리**: 에러 메시지 표시
* **재시도**: 사용자가 다시 버튼 클릭하여 재시도 가능
