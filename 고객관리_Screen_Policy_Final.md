# Screen Policy: 고객 관리

## ■ Screen Meta
* **SCREEN_NAME**: 고객 관리
* **화면 목적**: 마이페이지 내 기업 고객 목록 조회, 추가, 상세 확인 및 삭제 관리
* **화면 형태**: 단일 페이지 + 다이얼로그 (팝업)
* **진입 경로**: 마이페이지 좌측 메뉴 → 고객 관리
* **관련 화면**: 고객 추가 다이얼로그, 고객 정보 다이얼로그, 삭제 확인 다이얼로그

---

## ■ 화면 구조

```
[고객 관리 목록 화면]
  ├─ [검색 영역] 카테고리 Dropdown + 검색어 입력
  ├─ [목록 영역] 고객 목록 테이블 (체크박스 포함)
  │     ├─ 개별 삭제 버튼
  │     └─ [기업명] 클릭 시 → 고객 정보 다이얼로그
  ├─ [하단] 선택 삭제 버튼 + 고객 추가 버튼 + 페이지네이션
  │
  ├─ [삭제 전 안내] 삭제 확인 다이얼로그
  ├─ [고객 추가] 다이얼로그
  └─ [고객 정보] 다이얼로그 (상세 보기 + 수정)
```

---

## ■ Field Rule: 고객 관리 목록 화면

### [검색 카테고리 Dropdown]
* **필수 여부**: 선택
* **입력 규칙**: Dropdown 선택
* **선택 항목**: 전체 / 기업명 / 사업자등록번호 / 담당자명
* **기본값**: 전체

### [검색어 입력 필드]
* **필수 여부**: 선택
* **입력 규칙**: 자유 텍스트
* **UI 정책**:
  - Placeholder: "검색어를 입력해주세요"
  - 검색 아이콘 클릭 또는 Enter 시 검색 실행
  - 검색 결과 없음 시: "검색 결과가 없어요" 안내

### [고객 목록 테이블]
* **표시 항목**: 체크박스 / 기업명 / 사업자등록번호 / 당해연도 접수건수(건) / 당해연도 총 결제금액(₩) / 담당자명 / 담당자 연락처 / 기능(삭제)
* **당해연도 기준**: 가입일 기준 1년
* **[기업명] 클릭 시**: 고객 정보 다이얼로그 노출
* **정렬 기준**: 등록일 내림차순 (최근 등록 고객 상단), 수정 시 수정일 기준 재정렬
* **페이지당 표시 건수**: 20건씩 보기
* **페이지네이션**: 하단 페이지 번호 표시

### [체크박스]
* **전체 선택**: 헤더 체크박스 클릭 시 전체 선택/해제
* **개별 선택**: 행 좌측 체크박스 클릭
* **선택 시**: 하단 "총 N건 선택됨 선택 삭제" 표시

### [삭제 버튼 (개별)]
* **위치**: 각 행 우측
* **동작**: 삭제 전 안내 다이얼로그 노출

### [선택 삭제 버튼]
* **노출 조건**: 체크박스 1개 이상 선택 시 활성화
* **동작**: 삭제 전 안내 다이얼로그 노출

### [+ 고객 추가 버튼]
* **위치**: 목록 상단 우측
* **동작**: 고객 추가 다이얼로그 노출

## ■ Dialog: 고객 추가

### [화면 구성]
```
고객 추가

[기업 정보]
기업명                    사업자등록번호 (선택)         대표자명 (선택)
[기업명을 입력해주세요]     [사업자등록번호 10자리 입력]   [대표자명을 입력해주세요]

[담당자 정보]
담당자명 (선택)            담당자 연락처 (선택)          담당자 이메일 주소 (선택)
[담당자명을 입력해주세요]   [담당자 연락처를 입력해주세요] [담당자 이메일 주소를 입력해주세요]

[추가하기]
```

### [기업명]
* **필수 여부**: 필수
* **입력 규칙**: 텍스트 입력
* **길이 제한**: 최대 30자
* **중복 기준**: 중복 불가 (동일 기업명 존재 시 에러 처리)
* **UI 정책**:
  - Placeholder: "기업명을 입력해주세요(최대 30자)"
  - 미입력 시: "기업명을 입력해주세요" 에러 메시지 표시
  - 중복 시: "이미 등록된 기업명이에요, 다른 기업명을 입력해주세요" 에러 메시지 표시

### [사업자등록번호]
* **필수 여부**: 선택
* **입력 규칙**: 숫자 10자리
* **중복 기준**: 중복 불가 (동일 사업자등록번호 존재 시 에러 처리)
* **UI 정책**:
  - Placeholder: "사업자등록번호 10자리를 입력해주세요"
  - 10자리 입력 완료 시 자동 휴/폐업 조회 (추후 휴/폐업 케이스 화면 설계 예정)
  - 10자리 미만 시: "사업자등록번호 10자리를 입력해주세요" 에러 메시지 표시
  - 중복 시: "이미 등록된 사업자등록번호에요" 에러 메시지 표시

### [대표자명]
* **필수 여부**: 선택
* **입력 규칙**: 텍스트 입력
* **UI 정책**:
  - Placeholder: "대표자명을 입력해주세요"

### [담당자명]
* **필수 여부**: 선택
* **입력 규칙**: 텍스트 입력
* **UI 정책**:
  - Placeholder: "담당자명을 입력해주세요"

### [담당자 연락처]
* **필수 여부**: 선택
* **입력 규칙**: 숫자만 입력, "-" 제외
* **UI 정책**:
  - Placeholder: "담당자 연락처를 입력해주세요"

### [담당자 이메일 주소]
* **필수 여부**: 선택
* **입력 규칙**: 이메일 형식 (@ 포함)
* **자동완성**: 텍스트 입력 시 @naver.com, @gmail.com, @daum.net, @hanmail.net 자동완성 목록 표시 (회원가입과 동일 방식)
* **UI 정책**:
  - Placeholder: "담당자 이메일주소를 입력해주세요"
  - 형식 오류 시: "올바른 형식의 이메일 주소를 입력해주세요" 에러 메시지 표시

### [추가하기 버튼]
* **활성화 조건**: 기업명 입력 완료 시
* **동작**: 유효성 검증 → 성공 시 고객 목록에 추가 → 다이얼로그 닫힘
* **실패 시**: 해당 필드 하단 에러 메시지 표시

### [닫기(X) 버튼]
* **동작**: 입력 데이터 초기화 후 다이얼로그 닫힘

## ■ Dialog: 고객 정보 (상세 보기 / 수정)

### [화면 구성]
```
고객 정보

[기 간] 전체  최근 1개월  최근 6개월  최근 1년  기간 설정

접수·결제 집계 (25년 9월 20일 ~ 25년 10월 20일)
접수건수               결제금액
120건 접수             1,230,000원

[기업 정보]
기업명                    사업자등록번호 (선택)         대표자명 (선택)
삼성SDS                   123-24-34533               김삼성

[담당자 정보]
담당자명 (선택)            담당자 연락처 (선택)          담당자 이메일 주소 (선택)
김담당                    01023232211               damdangja@naver.com

[수정하기]
```

### [기간 탭]
* **선택 항목**: 전체 / 최근 1개월 / 최근 6개월 / 최근 1년 / 기간 설정
* **기본값**: 최근 1년
* **기간 설정 선택 시**: Range Date-Picker 노출
  - 입력 항목: 시작일 / 종료일
  - UI 형식: 캘린더 범위 선택 (시작일 ~ 종료일)
  - 선택 가능 범위: 당일 기준 최근 1년 이내 (1년 이전 날짜 선택 불가, 비활성화 처리)
  - 시작일이 종료일보다 늦을 수 없음
  - 선택 완료 시 해당 기간 기준으로 접수·결제 집계 갱신

### [접수·결제 집계]
* **표시 항목**: 접수건수 / 결제금액
* **표시 기준**: 선택된 기간 기준 자동 산출
* **수정 불가**: 읽기 전용

### [기업명]
* **필수 여부**: 필수
* **입력 규칙**: 텍스트 입력
* **길이 제한**: 최대 30자
* **중복 기준**: 중복 불가 (기존 등록된 다른 고객과 동일 기업명 불가)
* **UI 정책**: 수정 가능

### [사업자등록번호]
* **필수 여부**: 선택
* **입력 규칙**: 숫자 10자리
* **중복 기준**: 중복 불가 (기존 등록된 다른 고객과 동일 사업자번호 불가)
* **UI 정책**: 수정 가능, 10자리 입력 완료 시 휴/폐업 조회

### [대표자명]
* **필수 여부**: 선택
* **UI 정책**: 수정 가능

### [담당자명]
* **필수 여부**: 선택
* **UI 정책**: 수정 가능

### [담당자 연락처]
* **필수 여부**: 선택
* **UI 정책**: 수정 가능

### [담당자 이메일 주소]
* **필수 여부**: 선택
* **입력 규칙**: 이메일 형식
* **UI 정책**: 수정 가능, 이메일 자동완성 제공

### [수정하기 버튼]
* **활성화 조건**: 항상 활성화
* **동작**: 유효성 검증 → 성공 시 정보 저장 → 다이얼로그 닫힘 및 목록 갱신
* **실패 시**: 해당 필드 하단 에러 메시지 표시

### [닫기(X) 버튼]
* **동작**: 다이얼로그 닫힘 (변경 내용 미저장)

## ■ Dialog: 삭제 전 안내

### [개별 삭제 시 화면 구성]
```
삭제 전 안내

이 기업을 정말 삭제할까요?
삭제하면 다시 복구할 수 없어요

[아니요]  [삭제하기]
```

### [선택 삭제 시 화면 구성]
```
삭제 전 안내

선택한 N건의 기업을 정말 삭제할까요?
삭제하면 다시 복구할 수 없어요

[아니요]  [삭제하기]
```
* **N**: 선택된 체크박스 건수 자동 반영

### [아니요 버튼]
* **동작**: 다이얼로그 닫힘, 삭제 취소

### [삭제하기 버튼]
* **동작**: 해당 고객 삭제 → 목록에서 즉시 제거 → 다이얼로그 닫힘
* **삭제 처리**: 고객 정보만 삭제, 기존 접수 내역은 유지
* **복구 불가**: 삭제 후 복구 불가

## ■ 화면 결과

### [고객 추가 성공 시]
* 고객 추가 다이얼로그 닫힘
* 고객 목록 상단에 신규 고객 행 추가 (등록일 기준 최신순 정렬)

### [고객 정보 수정 성공 시]
* 고객 정보 다이얼로그 닫힘
* 목록의 해당 행 정보 갱신, 수정일 기준으로 정렬 재반영

### [고객 삭제 성공 시]
* 삭제 확인 다이얼로그 닫힘
* 목록에서 해당 행 즉시 제거
* 총 건수 갱신
